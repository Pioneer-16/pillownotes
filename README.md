# 枕书阁 · PillowNotes

一个轻量的本地读书笔记管理工具，支持自定义模板、多笔记本分类、卡片式展示，内置 AI 助手。

## ✨ 特性

- 📝 自定义卡片模板 — 自由组合字段，适配不同笔记场景
- 📚 多笔记本管理 — 按主题分类，独立模板
- 🔗 笔记引用 — 交叉引用和单向引用，构建知识网络
- 🎨 卡片式展示 — 书名、朝代、页码一目了然
- 🔍 全文搜索 — 快速定位笔记内容
- 📋 筛选功能 — 按字段精确过滤
- 🖼️ 图片粘贴 — 直接粘贴图片到笔记
- 📥 导入导出 — JSON 格式备份恢复，支持跨设备迁移组件配置
- 🔒 纯本地部署 — 数据在自己电脑上，无需注册
- 🤖 AI 助手 — 智能问答、笔记推荐、流式输出、引用查询

## 🤖 AI 助手

内置 AI 聊天助手，支持 OpenAI 兼容 API（如 MiMo、OpenAI、DeepSeek 等）。

### 功能

- **工具调用（Tool Calling）** — AI 自主查询笔记数据库，支持搜索、按笔记本获取、按模板字段筛选、查询引用关系等
- **流式输出** — 逐字渲染，实时显示 AI 回复
- **Markdown 渲染** — 支持标题、代码块、表格、列表、引用、链接等
- **聊天上下文** — 多轮对话，AI 记住之前的交流内容
- **笔记推荐** — AI 回复后展示相关笔记卡片，点击跳转按钮可定位到对应笔记

### 配置

在 AI 聊天侧栏的设置面板中填写：

| 配置项 | 示例 |
|--------|------|
| API 地址 | `https://api.openai.com/v1` |
| API 密钥 | `sk-...` |
| 模型 | `gpt-4o-mini` |

## 🔗 笔记引用

支持在笔记之间建立引用关系，构建知识网络。

### 引用类型

| 类型 | 说明 |
|------|------|
| 交叉引用 | 双向关联，双方笔记都显示引用按钮 |
| 单向引用 | 仅源笔记显示引用按钮，目标笔记不显示 |

### 使用方式

1. 编辑笔记时，在「引用笔记」区域搜索目标笔记
2. 选择引用类型（交叉/单向），点击搜索结果添加
3. 笔记卡片工具栏出现引用图标，点击展开引用列表
4. 点击引用卡片的跳转按钮可定位到对应笔记

### AI 引用查询

AI 助手可查询笔记的引用关系。例如：

- 「这篇笔记引用了哪些其他笔记？」
- 「帮我找和这篇相关的笔记」

## 📝 排版格式

### 段落缩进

笔记支持中文首行缩进，规则如下：

- **行首无空格** → 自动缩进（首行缩进两字符）
- **行首有空格** → 不缩进（普通段落）

示例：
```
 这是一段会自动缩进的文字。
这是一段不会缩进的文字（行首有空格）。
```

### 代码块

使用 ` ``` ` 包裹代码，支持代码块语法：

````
```
function hello() {
  console.log("Hello!");
}
```
````

- 代码块内保留原始格式，不做缩进处理
- 组件开启「代码样式」后，整个字段以等宽字体显示

### 其他语法

- **粗体**：`**文字**`
- *斜体*：`*文字*`
- 图片：`![描述](图片链接)`
- 表格：使用 `|` 分隔列

## 🧩 自定义组件

模板系统支持以下组件类型，可自由组合：

| 组件类型 | 说明 |
|----------|------|
| 文本域 | 多行文本，支持 Markdown、表格、代码块 |
| 单行输入 | 简短文本字段 |
| 下拉选择 | 预设选项，支持自动补全 |
| 数字 | 数值字段，支持自定义格式（如 P000） |
| 日期 | 日期选择器 |
| 链接 | URL 字段，支持多链接 |
| 评分 | 五星评分 |

组件配置完全模块化：自定义字段的搜索、筛选、排序、导出导入均自动适配。

## 🚀 快速开始

```bash
# 克隆项目
git clone https://github.com/Pioneer-16/pillownotes.git
cd pillownotes

# 安装依赖
npm install

# 启动服务
npm start
```

然后打开浏览器访问 http://localhost:3000

或者直接双击 `start.bat`（Windows）。

## ⚙️ 配置

复制 `.env.example` 为 `.env`，可自定义端口和访问密码：

```env
PORT=3000
AUTH_PASSWORD=your_password
```

## 🗂️ 项目结构

```
├── app/web/          # 前端 + 服务端
│   ├── index.html    # 页面结构
│   ├── app.js        # 前端逻辑
│   ├── style.css     # 样式
│   ├── server.js     # Node.js 服务端
│   └── db.js         # SQLite 数据库层
├── data/             # 数据目录（自动创建）
│   ├── _globals.json # 全局配置（组件、模板、笔记本顺序等）
│   ├── notes.db      # SQLite 笔记数据库（含笔记和引用关系表）
│   └── images/       # 图片存储
└── data/sample/      # 样例数据
```

## 🛠️ 技术栈

- 前端：原生 HTML/CSS/JS，无框架依赖
- 后端：Node.js + 原生 HTTP
- 存储：SQLite（better-sqlite3）
- AI：OpenAI 兼容 API + Tool Calling + SSE 流式输出
- Markdown：marked.js
